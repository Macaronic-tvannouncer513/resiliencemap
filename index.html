// How it works

From raw federal data
to actionable risk scores

A four-stage pipeline that any municipality can run with one command — no GIS department, no vendor contracts.

01 / INGEST
📥
Fetch hazard data
Automated ingestion of FEMA flood zones, USGS earthquake events, and NOAA storm alerts via public APIs.
02 / STORE
🗄️
PostGIS spatial store
All geometries stored in PostgreSQL + PostGIS with spatial indexes enabling millisecond intersection queries.
03 / SCORE
⚡
Compute risk scores
Weighted composite of flood, seismic, storm, and social vulnerability components per census tract.
04 / SERVE
🌐
REST API + map dashboard
FastAPI endpoints and interactive Leaflet.js map. Query by county FIPS, tract GEOID, or export GeoJSON.
// Data sources

Built on authoritative
federal datasets

FEMA

National Flood Hazard Layer

Special Flood Hazard Area polygons covering CONUS. Area-weighted intersection per census tract determines flood exposure score.

USGS

Earthquake Event Catalog

Real-time seismic events via FDSN web service. Magnitude-distance decay model scores seismic risk within 500km radius.

NOAA

NWS Storm Alerts (CAP)

Active watches, warnings and advisories. Severity-weighted scoring captures current storm exposure for each tract.

CDC

Social Vulnerability Index

ATSDR SVI scores incorporate income, housing, and demographic vulnerability factors that amplify physical hazard impact.
